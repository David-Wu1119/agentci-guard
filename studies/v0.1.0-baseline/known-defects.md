# Known v0.1.0 Defects

These defects were reproduced or confirmed by source inspection before the
v0.1.1 implementation began.

| ID  | Class              | Minimal condition                                                                                 | v0.1.0 behavior                                                 | Correct behavior                                                                       |
| --- | ------------------ | ------------------------------------------------------------------------------------------------- | --------------------------------------------------------------- | -------------------------------------------------------------------------------------- |
| B01 | Action entrypoint  | Invoke the JavaScript Action through its declared inputs.                                         | CLI help, exit 1, no SARIF, no outputs.                         | Scan through `action.yml` and honor inputs.                                            |
| B02 | False negative     | AI CLI in a `run:` step, such as `claude ...`.                                                    | No finding in a minimal shell-only workflow.                    | Recognize shell AI execution and shell capability.                                     |
| B03 | False negative     | Secret or token-like value declared in workflow-level `env`.                                      | The effective job/step environment is not modeled.              | Apply workflow → job → step precedence.                                                |
| B04 | False positive     | Workflow has several triggers, but an AI job is gated to `push`.                                  | The job is treated as reachable from every workflow trigger.    | Analyze the event subset that can reach the job.                                       |
| B05 | False positive     | `permissions` is absent.                                                                          | Absence is treated like an empty, read-only map.                | Represent the default as `unknown` unless an explicit default is configured.           |
| B06 | False negative     | `discussions: write` or discussion events.                                                        | Relevant permission/event coverage is missing.                  | Model discussion and discussion-comment events and the `discussions` scope.            |
| B07 | False negative     | Local or remote reusable workflow invoked with job-level `uses`.                                  | Reusable workflows are not analyzed.                            | Resolve local workflows with cycle detection; mark unresolved remote calls incomplete. |
| B08 | Misclassification  | Invalid workflow YAML.                                                                            | Emits a high-severity prompt-injection finding.                 | Emit a parse diagnostic, not a security rule.                                          |
| B09 | False positive     | Benign text contains the word `Python` near a pinned Claude action.                               | Shell-access rule can fire from broad serialized-step matching. | Base shell semantics on `run:` and explicit tool capability, not incidental prose.     |
| B10 | Configuration      | Explicit `--config` path does not exist.                                                          | Missing file is silently ignored.                               | Fail with a clear configuration error.                                                 |
| B11 | Location quality   | Any SARIF finding.                                                                                | Every result points to line 1.                                  | Report a meaningful job or step line.                                                  |
| B12 | Checkout semantics | Privileged workflow checks out attacker-controlled code through variants not matching one string. | Detection depends on a stale textual special case.              | Model checkout refs and untrusted event reachability explicitly.                       |
